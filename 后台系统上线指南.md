## 二、Supabase 配置步骤（约 10 分钟）

### 第 1 步：创建 Supabase 项目

1. 打开 https://supabase.com 注册/登录
2. 点击 **New Project**
3. 填写：
   - Name: `woneng-energy`
   - Database Password: 设一个强密码（记下来！）
   - Region: 选离你最近的（如 `Southeast Asia (Singapore)` 或 `Northeast Asia (Tokyo)`）
4. 点 **Create new project**，等待约 2 分钟

### 第 2 步：获取 API 密钥

项目创建后：
1. 左侧菜单 → **Project Settings**（齿轮图标）
2. 点击 **API**
3. 找到两个关键信息：
   ```
   Project URL:  https://xxxxxxxxxxxx.supabase.co
   anon public:  eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
   把这两个值复制下来。

### 第 3 步：填入网站配置文件

你需要修改**两个**配置文件（内容相同）：

**文件 1：** `admin/js/config.js`
**文件 2：** `js/supabase-config.js`

打开这两个文件，替换成你的真实值：
```javascript
window.SUPABASE_CONFIG = {
    url: "https://xxxxxxxxxxxx.supabase.co",        // ← 替换这里
    anonKey: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."  // ← 替换这里
};
```

> ⚠️ 两个文件的 url 和 anonKey 必须完全一致。

### 第 4 步：执行数据库 Schema

1. 在 Supabase 左侧菜单 → **SQL Editor**
2. 点 **New query**
3. 打开本地文件 `admin/database/schema.sql`，复制全部内容
4. 粘贴到 SQL Editor 中
5. 点 **Run** 执行

这会创建 5 张表：
- `customers` — 客户信息
- `inquiries` — 询盘记录
- `follow_ups` — 跟进记录
- `news` — 新闻文章
- `seo_pages` — SEO 页面配置

### 第 5 步：创建管理员账号

1. Supabase 左侧菜单 → **Authentication** → **Users**
2. 点 **Add user** → **Create new user**
3. 填写管理员邮箱和密码（这就是你登录后台的账号）
4. 勾选 **Auto Confirm User**（自动确认，否则无法登录）
5. 点 **Create user**

## 三、如何使用各功能模块

### 3.1 登录后台

1. 访问 https://kkwen369.github.io/woneng-energy-website/admin/
2. 输入你在第 5 步创建的邮箱和密码
3. 点 **Sign In**

### 3.2 客户管理

- **客户列表**：`/admin/customers.html`
  - 搜索、按状态/类型筛选
  - 点击客户名查看详情
  - 「+ 添加客户」手动录入
  - 「导出 CSV」批量导出数据

- **客户详情**：`/admin/customer-detail.html?id=xxx`
  - 编辑客户信息
  - 添加跟进记录（下次跟进时间、沟通内容）
  - 查看该客户关联的询盘

- **询盘管理**：`/admin/inquiries.html`
  - 查看网站前端提交的所有询盘
  - 更新状态：待处理 → 已回复 → 已报价 → 已关闭
  - 「转为客户」：一键将询盘转为正式客户

### 3.3 企业资讯管理

- **文章列表**：`/admin/news.html`
  - 新建文章、编辑、删除
  - 发布/取消发布
  - 按分类筛选

- **文章编辑**：`/admin/news-edit.html?id=xxx`
  - 标题、URL slug（自动生成）
  - 摘要、富文本内容（带工具栏）
  - SEO 设置：meta title / description / keywords
  - 保存草稿或直接发布

- **前端展示**：`/news.html`
  - 自动显示所有已发布文章
  - 点击文章查看详情

### 3.4 SEO 优化管理

- **SEO 配置**：`/admin/seo.html`
  - 为每个页面设置独立的 SEO 元数据
  - meta title（建议 50-60 字符）
  - meta description（建议 150-160 字符）
  - meta keywords
  - Open Graph 标题/描述/图片（社交分享预览）
  - canonical URL
  - 前端页面会自动加载这些配置并注入到 `<head>` 中

## 四、日常使用流程

### 收到新询盘时
1. 客户在网站填写询盘表单 → 自动存入 `inquiries` 表
2. 登录后台 → 询盘管理 → 查看详情
3. 回复客户后更新状态为「已回复」
4. 报价后更新为「已报价」
5. 确认为正式客户后点「转为客户」

### 发布企业资讯时
1. 后台 → 企业资讯 → 新建文章
2. 填写标题、内容、SEO 信息
3. 点发布
4. 前端 `/news.html` 自动更新

### 持续优化 SEO 时
1. 后台 → SEO 管理
2. 选择要优化的页面
3. 优化 meta title / description / keywords
4. 设置 Open Graph 图片（社交分享时显示的图）
5. 保存后前端自动生效

## 五、注意事项

1. **anonKey 是公开的** — 这是 Supabase 的设计，anonKey 本身是安全的（只允许通过 RLS 策略定义的操作）。不要把 `service_role` key 放到前端代码里。

2. **RLS 已启用** — schema.sql 已配置行级安全策略：
   - `inquiries` 表允许匿名 INSERT（网站访客可提交询盘）
   - 其他所有操作需要管理员登录
   - `customers`、`follow_ups`、`news`、`seo_pages` 表只有认证用户可操作

3. **配置修改后需推送** — 修改 `config.js` 后，需要提交并推送到 GitHub 才能在线上生效。可以告诉我帮你推送。

4. **Supabase 免费额度** — 免费版包含 500MB 数据库、5GB 带宽、50000 月活用户，对于企业独立站完全够用。

## 六、遇到问题怎么办

| 问题 | 解决方案 |
|------|----------|
| 登录后台提示 "Setup needed" | 还没配置 Supabase，按上面步骤 2-5 操作 |
| 登录后白屏 | 检查 `config.js` 的 url/anonKey 是否正确 |
| 询盘没有自动入库 | 检查 schema.sql 是否执行成功，检查 `inquiries` 表是否存在 |
| 新闻页不显示文章 | 确认文章状态是「已发布」而非「草稿」 |
| SEO 标签没生效 | 检查 `seo_pages` 表是否有对应页面的记录 |
